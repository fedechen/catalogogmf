Restrições

Condição restritiva. Regra.

Para:

- Regras para a execução das tarefas;

- Segurança;

- Orientação;\

Quando Usar:

- Sequenciar a execução de tarefas;

- Existem pré-requisitos entre as tarefas;

- Existem regras para execução das tarefas;

- Incentivo para execução de tarefas distintas;

- Criar restrições quanto ao número de tarefas executadas por um determinado período de tempo;

- Tornar as tarefas acessíveis a todos;

- Exigir cautela na execução da tarefa;

Comportamento esperado:

- O participante cumpre com os requisitos adotados no projeto.

Como Usar:

- Desbloqueando tarefas durante o processo;

- Criando regras para a execução das tarefas (tempo disponível, número de tentativas, etc.);

- Permitir um número máximo para erros;

- A tarefa só pode ser executada se as regras forem seguidas;

- Regras são cuidadosamente explicadas antes de cada tarefa;

- Regras por perfil de participante;

- Regras para gatilho de ações do sistema;

- Apresentação de regras simples, claras e intuitivas;

Exemplo de uso:

- As tarefas relacionadas ao projeto são divididas por tipos. O projeto possui a restrição que o participante precisa fazer um tipo de tarefa diferente sequencialmente. Exemplo: para assumir uma nova tarefa de programação, o participante precisa contribuir ao sistema de perguntas e respostas, ou fazer uma documentação.
